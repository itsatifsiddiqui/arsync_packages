1. Bump the version in pubspec.yaml appropriately based on the changes:
   - MAJOR: Breaking changes
   - MINOR: New features, backward compatible
   - PATCH: Bug fixes, dependency updates, minor improvements

2. Update CHANGELOG.md with the new version entry describing what changed
   - Keep the changelog entry short and concise: 3-5 lines maximum
   - Focus on the most important changes

3. Update the installation instructions in README.md to reference the new version

4. Show the user the updated version and changelog entry
